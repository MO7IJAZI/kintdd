# تثبيت وتفعيل إصلاح نظام الخطوط

## الملخص
تم إصلاح نظام الخطوط المخصصة بشكل كامل لضمان:
1. ✅ حفظ الخطوط في قاعدة البيانات (ليس ملف JSON)
2. ✅ تحميل الخطوط بشكل صحيح على الواجهة الأمامية
3. ✅ إمكانية تغيير الخط على النصوص المنسوخة
4. ✅ الحفاظ على أنماط النصوص المنسوخة مع إمكانية التعديل

---

## الخطوات المطلوبة للتثبيت

### 1️⃣ تطبيق هجرة قاعدة البيانات

```bash
npx prisma migrate deploy
```

أو إذا كنت تستخدم npm:
```bash
npm run prisma:migrate
```

**ماذا يفعل:** ينشئ جدول `custom_fonts` في قاعدة البيانات لتخزين بيانات الخطوط.

---

### 2️⃣ التحقق من مصدر البيانات

إذا كان لديك خطوط مخزنة في ملف `public/custom-fonts.json`، يمكنك نقل البيانات إلى قاعدة البيانات:

```bash
node scripts/migrate-fonts-to-db.js
```

**ملاحظة:** سيتم إنشاء هذا السكريبت إذا لم يكن موجوداً.

---

### 3️⃣ إعادة تشغيل الخادم

```bash
npm run dev
```

## كيفية الاستخدام

### إضافة خط جديد
1. في محرر النصوص الغني (RichTextEditor)، اضغط على "🔤 رفع خط مخصص"
2. ادخل اسم الخط (مثال: Cairo, Droid Arabic Kufi)
3. اختر ملف الخط (.ttf, .otf, .woff, .woff2)
4. سيتم رفع الخط وتخزينه في قاعدة البيانات تلقائياً

### استخدام الخط المخصص
1. اختر النص الذي تريد تطبيق الخط عليه
2. اذهب إلى القائمة العلوية → Font Family
3. اختر اسم الخط من القائمة
4. تم تطبيق الخط! ✅

### تغيير الخط على نص منسوخ
- الآن يمكنك تغيير خط أي نص منسوخ بدون مشاكل
- حتى لو كان النص الأصلي بخط مختلف
- فقط اختر النص وغير الخط من قائمة Font Family

### حذف خط
- اضغط على الـ ✕ بجانب اسم الخط في قائمة الخطوط المرفوعة
- سيتم حذف الخط من قاعدة البيانات والملفات

## استكشاف الأخطاء

### المشكلة: الخطوط لا تظهر بعد الرفع
**الحل:**
1. تحقق من أن الهجرة تم تطبيقها: `npx prisma migrate status`
2. تأكد من أن ملف الخط بصيغة صحيحة (.ttf, .otf, etc.)
3. تحقق من أن الخادم يعمل بشكل صحيح
4. افتح الـ Browser Console وتحقق من الأخطاء

### المشكلة: لا يمكن تغيير الخط على نص منسوخ
**الحل:**
1. تأكد من أنك تستخدم أحدث نسخة من الكود
2. جرب تحديث الصفحة (Ctrl + Shift + R)
3. تأكد من أن الخط المراد التطبيق موجود في قائمة الخطوط

### المشكلة: خطأ في قاعدة البيانات أثناء الهجرة
**الحل:**
```bash
# تحقق من الاتصال بقاعدة البيانات
npx prisma db push

# أو أعد محاولة الهجرة
npx prisma migrate resolve --rolled-back 20260423_add_custom_fonts_table
npx prisma migrate deploy
```

## ملاحظات تقنية

### معمارية النظام
```
Frontend (RichTextEditor)
    ↓
API /api/fonts (POST)
    ↓
fontManager.ts (saveCustomFont)
    ↓
Prisma ORM
    ↓
قاعدة البيانات MySQL/MariaDB
    ↓
ملف الخط (public/uploads/fonts/)
```

### تدفق رفع الخط
1. المستخدم يختار ملف الخط في محرر النصوص
2. يتم رفع الملف عبر `/api/upload`
3. يتم حفظ الملف في `public/uploads/fonts/`
4. يتم حفظ البيانات الوصفية في `custom_fonts` جدول
5. يتم استرجاع الخطوط النشطة من قاعدة البيانات
6. يتم توليد CSS @font-face وتطبيقه على الـ editor

### أمان البيانات
- جميع عمليات POST و DELETE محمية بـ Auth
- يتم التحقق من أن المستخدم authenticated
- يتم حفظ البيانات الوصفية كاملة في قاعدة البيانات
